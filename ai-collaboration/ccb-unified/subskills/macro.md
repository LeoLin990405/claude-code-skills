# CCB-Macro - 分布式宏观研究（8 AI 串行协作）

## 概述

使用 CCB 的 8 AI 分布式架构进行中国 A 股市场的日常宏观研究，每个 AI 负责特定研究领域，串行执行确保可靠性。

## 串行执行模式（强制）

```
Gemini → Codex → OpenCode → iFlow → Kimi → Qwen → DeepSeek → Claude
 (基本面)  (技术)   (估值)    (资金)  (行业)  (国际)   (政策)   (舆情+报告)

每个 AI 完成后才启动下一个
每 30 秒轮询检查完成状态
```

**优势**:
- ✅ 避免资源竞争
- ✅ 确保结果可靠性
- ✅ 便于追踪和调试
- ✅ 减少任务丢失

## 实现（使用 CCB + Gateway API）

### 串行执行流程

```bash
#!/bin/bash
# macro_research_serial.sh

DATE=$(date +%Y-%m-%d)
echo "🔍 开始宏观研究: $DATE"

# 研究员列表（按执行顺序）
RESEARCHERS=(
    "gemini:基本面研究员"
    "codex:技术面研究员"
    "opencode:估值研究员"
    "iflow:资金流研究员"
    "kimi:行业研究员"
    "qwen:国际宏观研究员"
    "deepseek:政策研究员"
)

declare -A RESULTS

# 串行执行每个研究员
for researcher in "${RESEARCHERS[@]}"; do
    IFS=':' read -r provider role <<< "$researcher"

    echo ""
    echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
    echo "📊 启动: $role ($provider)"
    echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"

    # 加载任务模板
    TASK_FILE="~/.local/share/codex-dual/assets/task_templates/${provider}_macro.txt"

    if [ -f "$TASK_FILE" ]; then
        PROMPT=$(cat "$TASK_FILE")
    else
        PROMPT="作为${role}，分析今日市场（${DATE}）"
    fi

    # 提交任务
    ID=$(ccb-submit $provider "$PROMPT")
    echo "✓ Request ID: $ID"

    # 轮询等待完成
    echo "⏳ 等待结果..."
    MAX_ATTEMPTS=12  # 最多等待 60 秒
    ATTEMPT=0

    while [ $ATTEMPT -lt $MAX_ATTEMPTS ]; do
        sleep 5
        ((ATTEMPT++))

        if RESULT=$(ccb-query get $ID 2>/dev/null); then
            echo "✅ $role 完成 (${ATTEMPT}次轮询)"
            RESULTS[$provider]=$RESULT

            # 保存中间结果
            echo "$RESULT" > "results/${provider}_${DATE}.md"
            break
        else
            echo "  ⏳ 仍在处理... ($ATTEMPT/$MAX_ATTEMPTS)"
        fi
    done

    if [ $ATTEMPT -eq $MAX_ATTEMPTS ]; then
        echo "⏱️  $role 超时"
        RESULTS[$provider]="[超时未完成]"
    fi
done

# Claude 执行自己的舆情研究
echo ""
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "🔵 Claude 舆情研究"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"

# Claude 的舆情研究（使用 WebSearch tool）
# [执行舆情研究]

# 生成最终报告
echo ""
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "📝 生成最终报告"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"

cat > "macro_report_${DATE}.md" <<EOF
# A股宏观研究报告

**日期**: $DATE
**研究方法**: 8-AI 串行分布式协作
**搜索次数**: 200+

---

## 📊 投资决策摘要

[Claude 整合所有 8 个研究员的观点，生成投资决策]

---

## 💰 模拟投资组合

[组合详情]

---

## 🤖 多 Agent 共识

### 研究员观点

$(for provider in gemini codex opencode iflow kimi qwen deepseek; do
    echo "#### $provider"
    echo "${RESULTS[$provider]}"
    echo ""
done)

---

## 📈 市场回顾

[详细市场分析]

---

**生成时间**: $(date)
EOF

echo "✅ 报告已生成: macro_report_${DATE}.md"
```

## 最佳实践

1. **串行执行** - 确保每个 AI 完成后再启动下一个
2. **轮询检查** - 每 30 秒检查一次状态
3. **保存中间结果** - 每个 AI 的输出单独保存
4. **状态追踪** - 维护执行状态表
5. **投资决策在前** - 报告结构把决策摘要放最前面
6. **详细调仓理由** - 引用具体研究结论
7. **记录历史** - 追踪组合表现

## 与 macro-research-ccb skill 的关系

| 特性 | macro-research-ccb | ccb-macro |
|------|-------------------|----------|
| AI 数量 | 8 | 8（相同） |
| 执行模式 | 串行 | 串行（相同） |
| 调用方式 | 旧 CLI | ccb-submit（统一） |
| 轮询机制 | 手动 | 自动化脚本 |
| 状态追踪 | 简单 | 完整追踪表 |
| Subagent | 不支持 | 可选集成 |
| 监控 | 无 | Gateway 监控 |
